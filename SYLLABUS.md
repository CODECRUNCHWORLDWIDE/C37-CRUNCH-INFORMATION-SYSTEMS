# C37 · Crunch Information Systems — Syllabus

**Format:** 12 weeks · self-paced · open-source (GPL-3.0). Each week = 3 lectures, 3 exercises, 2 challenges, a mini-project, homework, and a quiz.

**Prerequisites:** You can run commands in a terminal and write a little Python. No prior systems, database, or cloud experience assumed — this course goes from zero to expert. [C33 Crunch SQL](../C33-CRUNCH-SQL/) is a helpful companion but not required.

**Assessment (honor-based):** tick each lesson complete in the reader; finish the weekly mini-project and quiz. A certificate is issued on 100% completion to One-Stop members.

**Data rule:** every data store, model, and query in this course uses SQL (PostgreSQL, SQLite fallback) and/or Python (pandas). We never use a spreadsheet as a database — even for finance-style workflows — because a shared, queryable, constrained store is what real systems need. Spreadsheets are covered in C41 Crunch Excel.

| Week | Theme | Mini-project |
|------|-------|--------------|
| 1 | Information systems foundations — people, process, data, technology; IS value | Map a real business as an information-systems diagram |
| 2 | Systems analysis + requirements — stakeholders, use cases, user stories | Write a requirements spec + use cases for a real need |
| 3 | Data modeling + databases — ER modeling, normalization, SQL schema | Design and build a normalized schema in PostgreSQL |
| 4 | Querying data with SQL + Python (pandas) — joins, aggregation, pipelines | Build a repeatable SQL→pandas reporting pipeline |
| 5 | Process modeling + automation — BPMN, workflow automation in code | Automate a manual multi-step business workflow |
| 6 | Enterprise systems — ERP/CRM concepts, master data, data flows | Model core ERP/CRM entities + master-data flows in SQL |
| 7 | Integration + APIs — REST, webhooks, ETL/ELT between systems | Integrate two systems with an API + ETL job |
| 8 | Cloud architecture — IaaS/PaaS/SaaS, deployment, availability, scaling | Deploy the system to the cloud with a scaling plan |
| 9 | Security, privacy + governance — access control, privacy, compliance | Add auth, access control, and a governance policy |
| 10 | Analytics + business intelligence — warehouse, KPIs, dashboards | Build a BI dashboard on a real data warehouse |
| 11 | AI integration — embedding ML/LLMs as responsible decision support | Add an AI decision-support feature to the system |
| 12 | Capstone — design a full intelligent information system for an org | Design, cost, and defend an end-to-end system |

**Outcome:** analyze an organization, model and store its data in SQL, automate its processes, integrate and deploy its systems in the cloud, govern and secure them, and layer analytics and AI on top — then design and defend a complete intelligent information system.
